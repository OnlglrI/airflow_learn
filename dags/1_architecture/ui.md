## UI / Скриншоты

Пользовательский интерфейс Airflow (Airflow UI) упрощает мониторинг и отладку ваших конвейеров данных. Ниже приведён краткий обзор некоторых возможностей и визуализаций, доступных в Airflow UI.

---

## DAGs View

Список DAG’ов в вашем окружении и набор быстрых переходов к полезным страницам. С первого взгляда можно увидеть, сколько задач завершились успешно, с ошибкой или выполняются в данный момент.
Чтобы скрыть завершённые задачи, установите параметр `show_recent_stats_for_completed_runs = False`.

Для фильтрации DAG’ов (например, по командам) можно добавлять теги в каждый DAG. Фильтр сохраняется в cookie браузера и может быть сброшен кнопкой **Reset**. Например:

```python
dag = DAG("dag", tags=["team1", "sql"])
```

![dags](../../img/ui/dags.png)

## Datasets View

Объединённый список текущих датасетов и граф, иллюстрирующий, как они создаются и потребляются DAG’ами.

При нажатии на любой датасет в списке или на графе он и связанные с ним зависимости будут подсвечены, а список будет отфильтрован так, чтобы показать недавнюю историю task instance’ов, которые обновляли этот датасет, а также информацию о том, запускались ли вследствие этого новые DAG Run’ы.

![datasets](../../img/ui/datasets.png)

---

## Grid View

Столбчатая диаграмма и табличное представление DAG’а, растянутое во времени. В верхней строке отображается диаграмма DAG Run’ов по длительности, ниже — task instance’ы. Если пайплайн выполняется с задержкой, можно быстро увидеть, на каком этапе возникла проблема и какие шаги блокируют выполнение.

![grid](../../img/ui/grid.png)

Панель деталей обновляется при выборе DAG Run’а нажатием на столбец с длительностью:

![grid\_run\_details](../../img/ui/grid_run_details.png)

Или при выборе task instance нажатием на ячейку со статусом:

![grid\_instance\_details](../../img/ui/grid_instance_details.png)

Ручные запуски помечаются иконкой воспроизведения (аналогично кнопке **Trigger DAG**):

![task\_manual\_vs\_scheduled](../../img/ui/task_manual_vs_scheduled.png)

Группы задач обозначаются значком-стрелкой и могут быть свернуты или развернуты:

![grid\_task\_group](../../img/ui/grid_task_group.png)

Mapped Tasks обозначаются квадратными скобками и отображают таблицу всех экземпляров сопоставленных задач в панели **Mapped Tasks**:

![grid\_mapped\_task](../../img/ui/grid_mapped_task.png)

## Graph View

Graph View — одно из самых информативных представлений. Оно позволяет визуализировать зависимости DAG’а и их текущее состояние для конкретного запуска.

![graph](../../img/ui/graph.png)

---

## Calendar View

Calendar View предоставляет обзор всей истории DAG’а за месяцы или даже годы, позволяя быстро увидеть тренды общего уровня успешных и неуспешных запусков с течением времени.

![calendar](../../img/ui/calendar.png)

---

## Variable View

Variable View позволяет просматривать, создавать, редактировать и удалять пары ключ–значение переменных, используемых во время выполнения задач.
Значение переменной по умолчанию скрывается, если ключ содержит одно из слов:
`password`, `secret`, `passwd`, `authorization`, `api_key`, `apikey`, `access_token`,
однако это поведение можно настроить для отображения значения в открытом виде. См. раздел *Masking sensitive data*.

![variable\_hidden](../../img/ui/variable_hidden.png)

---

## Gantt Chart

Диаграмма Ганта позволяет анализировать длительность выполнения задач и их перекрытие во времени. С её помощью можно быстро выявить узкие места и определить, на какие этапы приходится основная часть времени выполнения конкретных DAG Run’ов.

![gantt](../../img/ui/gantt.png)

---

## Task Duration

Отображает длительность выполнения различных задач за последние N запусков. Это представление позволяет находить выбросы и быстро понимать, где именно тратится время в DAG’е на протяжении множества запусков.

![duration](../../img/ui/duration.png)

---

## Landing Times

Landing time для task instance — это разница между окончанием интервала данных DAG Run’а (то есть моментом, когда DAG «должен» был выполниться) и фактическим временем завершения task instance.

![landing\_times](../../img/ui/landing_times.png)

---

## Code View

Прозрачность — это важно. Хотя код вашего пайплайна хранится в системе контроля версий, этот экран предоставляет быстрый доступ к коду, который генерирует DAG, и даёт дополнительный контекст при анализе его работы.

![code](../../img/ui/code.png)
